# Open PR

## Overview
Final phase of a swe-team run. Pushes the branch and opens a PR against `base_branch` with a structured, evidence-backed body so the human reviewer can audit the run without spelunking JSONL files. After PR creation, the run is considered `succeeded` and the `runs/current` symlink is removed (by init/cleanup). Failing to open a PR is NOT a partial success — either the PR exists or the run is `failed`.

## When to Use
- `swe-pr` spawned at end of VERIFY phase after whole-PR sem sweep returned clean.
- All tasks have both mech+sem `verified:true` entries in `verification.jsonl`.
- `git log <base>..HEAD` is non-empty.

## When NOT to Use
- Any task still `pending` or `in_progress` — BUILD isn't finished.
- VERIFY phase hasn't emitted its whole-PR verdict — don't ship unverified.
- `require_clean_working_tree` in config is true and `git status --porcelain` is non-empty.
- `gh` CLI not authenticated — fail fast with `run_abort` instead of a half-open PR.

## Process
1. Preconditions.
   ```bash
   command -v gh >/dev/null || { echo "gh not found"; exit 1; }
   gh auth status >/dev/null 2>&1 || { echo "gh not authed"; exit 1; }
   BASE=$(jq -r .base_branch .claude/swe-team/runs/current/run.json)
   BRANCH=$(jq -r .branch .claude/swe-team/runs/current/run.json)
   [ "$(jq -r .gh.require_clean_working_tree swe-team.config.json)" = "true" ] && \
     [ -n "$(git status --porcelain)" ] && { echo "working tree dirty"; exit 1; }
   ```
2. Assemble PR body sections into `/tmp/pr-body.md`:
   ```bash
   {
     echo "## Summary"
     jq -r '.requirement.text // .requirement.fetched_content' \
       .claude/swe-team/runs/current/run.json | head -n 20

     echo; echo "## Changes"
     jq -r '.tasks[] | "- **\(.id)** [\(.size)] \(.title)"' \
       .claude/swe-team/runs/current/tasks.json

     echo; echo "## Plan"
     jq -r '.tasks[] | "### \(.id): \(.title)\n\(.description)\n\nAcceptance:\n\(.acceptance | map("- " + .) | join("\n"))\n"' \
       .claude/swe-team/runs/current/tasks.json

     echo; echo "## Verification"
     jq -r 'select(.kind=="verification") |
       "- \(.task_id) \(.tier) verified=\(.verified) commit=\(.evidence.commit_sha[0:7])"' \
       .claude/swe-team/runs/current/verification.jsonl

     echo; echo "## Breaking changes"
     echo "None auto-detected. Reviewer: confirm."

     echo; echo "## Commits"
     git log --oneline "$BASE"..HEAD

     echo; echo "---"
     echo "Generated by swe-team v$(cat .claude/swe-team/VERSION 2>/dev/null || echo unknown)."
     echo "Run ID: $(jq -r .run_id .claude/swe-team/runs/current/run.json)"
     echo "Review the \`.claude/swe-team/runs/<run-id>/\` directory for full evidence."
   } > /tmp/pr-body.md
   ```
3. Push the branch.
   ```bash
   git push -u origin "$BRANCH"
   ```
4. Create the PR.
   ```bash
   TITLE=$(jq -r '.tasks[0].title' .claude/swe-team/runs/current/tasks.json \
     | sed 's/^/swe-team: /')
   LABELS=$(jq -r '.gh.pr_labels | join(",")' swe-team.config.json)
   DRAFT=""
   [ "$(jq -r .gh.pr_draft swe-team.config.json)" = "true" ] && DRAFT="--draft"
   PR_URL=$(gh pr create \
     --base "$BASE" \
     --head "$BRANCH" \
     --title "$TITLE" \
     --body-file /tmp/pr-body.md \
     --label "$LABELS" \
     $DRAFT \
     | tail -n 1)
   ```
5. Persist PR metadata.
   ```bash
   jq -n --arg url "$PR_URL" --arg branch "$BRANCH" \
     '{url:$url, branch:$branch}' \
     > .claude/swe-team/runs/current/pr.json
   jq --arg s "succeeded" '.status = $s' \
     .claude/swe-team/runs/current/run.json > /tmp/r.json && \
     mv /tmp/r.json .claude/swe-team/runs/current/run.json
   ```
6. Emit `run_complete`.
   ```bash
   jq -nc \
     --arg rid "$(jq -r .run_id .claude/swe-team/runs/current/run.json)" \
     --arg ts "$(date -u +%FT%TZ)" \
     --arg url "$PR_URL" --arg branch "$BRANCH" \
     '{kind:"run_complete", ts:$ts, run_id:$rid, agent:"swe-pr",
       pr_url:$url, branch:$branch}' \
     >> .claude/swe-team/runs/current/events.jsonl
   ```

## Anti-Rationalizations
| Excuse | Rebuttal |
|---|---|
| "One task is missing sem verified but the others are fine — ship it." | Every task requires mech+sem verified. The phase-exit hook will block you anyway. |
| "I'll push --force to clean up the branch." | `guard-destructive-git.sh` blocks `push --force`. Squash/rebase is out of scope for MVP. |
| "Skip the Verification section; the reviewer can read verification.jsonl." | PR body is the evidence summary. Omitting it defeats the human-gate design. |
| "I'll open a draft PR to avoid labels." | Draft vs labels are orthogonal — both are config-driven. Respect `gh.pr_draft` and `gh.pr_labels`. |
| "No commits since base — PR body is short but fine." | An empty-diff PR is nonsense. If git log is empty, emit `run_abort` with reason `no_commits`. |
| "gh failed; I'll write the URL by hand." | If `gh` fails, the PR doesn't exist. No PR = no `run_complete`. Fail honestly. |

## Red Flags
- `git push` fails (permission, protected branch) — the branch policy is wrong; `run_abort` with reason.
- `gh pr create` returns non-zero — don't synthesize a URL; fail.
- PR body > 65KB — GitHub truncates; shorten Verification section to just task/tier/verified.
- `pr.json.url` does not match the URL `gh` printed — state corruption.
- `run.json.status` is already `succeeded` before this skill — duplicate invocation; bail.

## Verification
After this skill completes:
- `pr.json` exists in run dir with `url` and `branch` populated.
- `run.json.status == "succeeded"`.
- One `run_complete` event in `events.jsonl` with `pr_url` and `branch` populated (per `event-schema.json`).
- `gh pr view "$PR_URL" --json state` returns a live PR.
- The remote branch exists: `git ls-remote --heads origin "$BRANCH"` non-empty.
- PR body contains the 7 sections: Summary, Changes, Plan, Verification, Breaking changes, Commits, Footer.
